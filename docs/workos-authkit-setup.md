# WorkOS AuthKit Setup

Last updated: 2026-05-09

The cloud admin uses **WorkOS AuthKit + Convex**. For this product we want **Google sign-in only**.

Official docs used:

- Convex AuthKit: https://docs.convex.dev/auth/authkit/
- Convex AuthKit existing app setup: https://docs.convex.dev/auth/authkit/add-to-app
- Convex AuthKit automatic config: https://docs.convex.dev/auth/authkit/auto-provision
- WorkOS Social Login: https://workos.com/docs/authkit/social-login

## What Is Already Configured

- `convex.json` configures AuthKit for local dev at `http://localhost:3000`.
- `convex/auth.config.ts` validates WorkOS-issued JWTs in Convex.
- `apps/cloud-admin/src/proxy.ts` enables AuthKit request protection for Next.js.
- `apps/cloud-admin/src/app/callback/route.ts` handles the WorkOS callback.
- `apps/cloud-admin/src/app/sign-in/route.ts` redirects to hosted AuthKit.
- `apps/cloud-admin/src/components/convex-client-provider.tsx` bridges WorkOS AuthKit tokens into Convex.
- `convex/viewer.ts` verifies Convex can see the authenticated identity.

## Where To Put Envs

Put local development secrets in the root `.env.local` file:

```bash
# Already generated by convex dev
CONVEX_DEPLOYMENT=dev:...
CONVEX_URL=https://....convex.cloud
CONVEX_SITE_URL=https://....convex.site

# WorkOS AuthKit
WORKOS_CLIENT_ID=client_...
WORKOS_API_KEY=sk_test_...
WORKOS_COOKIE_PASSWORD=replace-with-output-of-openssl-rand-base64-24
NEXT_PUBLIC_WORKOS_REDIRECT_URI=http://localhost:3000/callback

# Next.js client-side Convex URL
NEXT_PUBLIC_CONVEX_URL=https://....convex.cloud

# Hub-to-Convex sync HTTP action
CONVEX_HTTP_URL=https://....convex.site
POS_SYNC_SECRET=use-a-long-random-value

# Local hub bootstrap admin device token
HUB_ADMIN_TOKEN=use-a-long-random-value
```

Generate `WORKOS_COOKIE_PASSWORD`:

```bash
openssl rand -base64 24
```

For production, put equivalent production values in the hosting provider environment, not in source control:

- `WORKOS_CLIENT_ID`
- `WORKOS_API_KEY`
- `WORKOS_COOKIE_PASSWORD`
- `NEXT_PUBLIC_WORKOS_REDIRECT_URI`
- `NEXT_PUBLIC_CONVEX_URL`
- `CONVEX_DEPLOY_KEY`
- `CONVEX_HTTP_URL`
- `POS_SYNC_SECRET`
- `HUB_ADMIN_TOKEN`

## Google-Only AuthKit

Provider enablement is controlled in the WorkOS Dashboard, not this repo.

In WorkOS:

1. Go to Authentication/AuthKit provider settings.
2. Configure and enable **Google** social login.
3. Disable other login methods you do not want exposed:
   - Email/password
   - Magic Auth / email OTP
   - Microsoft, GitHub, Apple, etc.
4. Keep redirect URI as `http://localhost:3000/callback` for local dev.
5. Keep CORS origin as `http://localhost:3000` for local dev.

WorkOS docs note that social providers appear on the AuthKit page after they are configured and enabled.

## Commands

Because `convex.json` enables Convex-managed AuthKit provisioning, the first AuthKit run must happen in an
interactive terminal:

```bash
pnpm exec convex dev
```

Convex will ask you to associate/create a WorkOS team for automatic AuthKit environment provisioning. This cannot run
from a non-interactive agent shell.

After WorkOS provisioning and envs are present:

```bash
pnpm exec convex dev
pnpm dev:cloud
```

Open:

```text
http://localhost:3000
```

Click **Sign in with Google**. After the callback, the page should show the authenticated admin state. The backend
identity smoke-test function is `convex/viewer.ts`; after `pnpm exec convex dev` regenerates `convex/_generated`, it can
be wired into the page or called from client code as `api.viewer.get`.

If you do not want Convex-managed WorkOS provisioning, remove the `authKit.dev` section from `convex.json` and set
`WORKOS_CLIENT_ID` plus `WORKOS_API_KEY` manually in the Convex dashboard and `.env.local`.
